\section{Технический проект}

\subsection{Общая характеристика организации решения задачи}

Необходимо спроектировать и разработать оптимизирующий компилятор для языка Common Lisp, который способен транслировать исходное S-выражение в байт-код для целевой виртуальной машины. Компилятор должен поддерживать основные конструкции, а также применять ряд оптимизаций для повышения эффективности генерируемого кода.

Компилятор представляет собой программную систему, которая принимает на вход S-выражение и флаги оптимизации, преобразует выражение в промежуточное представление, применяет оптимизации и генерирует байт-код.

\subsection{Архитектура оптимизирующего компилятора Common Lisp}

Компилятор должен быть представлен в виде набора модулей, каждый из которых отвечает за ту или иную фазу компиляции. Должны быть определены следующие модули:

\begin{enumerate}
\item \texttt{common}. Включает в себя различные вспомогательные функции, которые используются другими модулями, но при этом тематически не подходят ни к одному из них.
\item \texttt{macro}. Включает в себя две основные функции -- \texttt{(make-macro list)}, которая регистрирует новый макрос, используя переданные через список \texttt{list} название, параметры и тело макроса, и \texttt{(macroexpand args vals body)}, которая раскрывает макрос с телом \texttt{body}, заменяя символы \texttt{args} значениями \texttt{vals}, а также другие вспомогательные функции. Функция \texttt{macroexpand} возвращает S-выражение как результат раскрытого макроса, либо вызывает исключение в случае ошибки.
\item \texttt{compiler}. Состоит из функции \texttt{(compile expr)}, которая принимает на вход исходное S-выражение \texttt{expr} и производит его статический анализ, а также внутренние вспомогательных функциий для анализа отдельных подвыражений. Встречая определение макроса или его раскрытие, \texttt{compile} вызывает функции из модуля \texttt{macro}. Функция \texttt{compile} возвращает упрощённую промежуточную форму искомого S-выражения в случае успеха и вызывает исключение в случае ошибки.
\item \texttt{optimizer}. Состоит из функции \texttt{(optimize-tree tree flags)}, которая принимает на вход промежуточную форму S-выражения \texttt{tree}, полученную на выходе из функции \texttt{compile}, и преобразует данную форму, применяя к ней оптимизации, указанные в качестве списка флагов оптимизаций \texttt{flags}, а также вспомогательных функций, отвечающих за конкретные оптимизации. Функция \texttt{opimize-tree} возвращает промежуточную форму с применёнными к ней оптимизациями, которая может остаться без изменений, если ни одна из указанных оптимизаций неприменима к данной промежуточной форме, либо если в качестве списка флагов оптимизаций на вход функции передан пустой список.
\item \texttt{generator}. Состоит из функции \texttt{(generate tree)}, которая принимает на вход оптимизированную промежуточную форму S-выражения \texttt{tree}, полученную на выходе из функции \texttt{optimize}, и на её основе генерирует код ассемблера, состоящий из мнемоник инструкций, их операторов и меток для инструкций перехода, а также вспомогательных функций для генерации отдельных элементов промежуточной формы. Функция \texttt{generate} возвращает массив мнемоник инструкций, их операндов и меток.
\item \texttt{assembler}. Состоит из функции \texttt{(assemble code)}, которая принимает на вход ассемблер код -- массив мнемоник инструкций, операндов инструкций и меток для перехода, полученный на выходе из функции \texttt{generate}, и переводит его в байт-код для виртуальной машины, а также других вспомогательных функций. Функция \texttt{assemble} возвращает массив байтов, составляющих искомый байт-код, список констант, используемых в программе, размер памяти для глобальных переменных и таблицу названий функций и их адресов в случае успешного ассемблирования, а в случае ошибки -- вызывает исключение.
\end{enumerate}

\subsection{Фаза 1. Макроподстановка}

На данном этапе компиляции в исходном S-выражении регистрируются и раскрываются макросы. Макроподстановка представляет собой интерпретацию ограниченного подмножества Lisp, вычисляя в теле макроса всё, что можно во время компиляции. Это сохраняет выразительную силу макросов и в то же время является значительной оптимизацией компилятора.

Форма \texttt{defmacro} добавляет новый макрос с указанным именем, числом обязательных аргументов, списком параметров и телом в специальный список. Если макрос принимает фиксированное число аргументов, то он попадает в список \texttt{*fix-macros*}. Если макрос принимает переменное число аргументов, то он попадает в список \texttt{*nary-macros*}. В результате добавления нового макроса выражение \texttt{defmacro} исчезает из исходного S-выражения.

Вызов макроса состоит из поиска аргументов и тела соответствующего макроса, сопоставления фактических значений символам аргументов и последующего рекурсивного раскрытия тела макроса.

\subsubsection{Атомарные выражения и окружение макросов}

Константы (все атомарные значения, кроме символов) в процессе раскрытия остаются без изменений.

Для подстановки символов происходит симуляция окружения переменных, представляющее из себя список пар символов и значений. В начале раскрытия макроса окружение состоит из аргументов и их значений, оставленных в изначальном состоянии. Если какой-либо аргумент представлен в виде сложного выражения (специальная форма или применение примитива), то он остаётся в таком виде до тех пор, пока не понадобится его значение, в случае чего он будет вычислен.

В процессе раскрытия макроса окружение может быть расширено. В этом случае в начало списка окружения добавляются новые пары символов и их значений, а при восстановлении окружения -- удаляются из начала.

Также должна быть представлена хеш-таблица \texttt{*macro-globals*}, в которой хранятся глобальные переменные, также представленные в виде пар символов и значений. Эти глобальные переменные существуют только на данном этапе компиляции и используются для сохранения значений между различными вызовами макросов.

При подстановке символа происходит поиск пары с соответствующим символом сначала в текущем окружении, а затем в таблице глобальных переменных, в следствие чего символ заменяется своим значением. Если такого символа не существует ни в окружении, ни среди глобальных переменных, то возвращается ошибка компиляции. Символы \texttt{T} и \texttt{NIL} являются особым случаем: значение символа \texttt{T} -- тот же символ \texttt{T}, символа \texttt{NIL} -- пустой список.

Примеры:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defmacro m () 5)
  (m))
->
(progn
  5)

(progn
  (defmacro m () t)
  (m))
->
(progn
  T)

(progn
  (defmacro m (x) x)
  (m 5))
->
(progn
  5)
\end{lstlisting}
\end{figure}

\subsubsection{Вызов примитива}

Для вызова примитива вычисляются его аргументы, после чего происходит применение примитива к полученным значениями аргументов и возврат значения примитива. Если аргумент примитива -- символ, происходит подстановка символа, чтобы получить значение аргумента. Если аргумент сам является вызовом примитива, либо представлен в виде специальной формы, то происходит рекурсивное вычисление аргумента. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defmacro m (x) (+ x 1))
  (m 2))
->
(progn
  3)
\end{lstlisting}
\end{figure}

\subsubsection{Пользовательские функции}

Определения функций вне макросов регистрируются в специальные списки в зависимости от переменности числа аргументов, аналогично макросам (\texttt{*fix-functions*} и \texttt{*nary-functions*}). Определения функций внутри макросов запрещены.

Если в процессе раскрытия макроса встречается вызов функции, определённой к этому моменту пользователем, то происходит вычисление аргументов, расширение окружения полученными значениями аргументов и раскрытие тела функции с расширенным окружением и с последующим возвратом вычисленного значения функции. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defun list (&rest args) args)
  (defmacro m (f) (list f 1 2 3))
  (m '+))
->
(progn
  (defun list (&rest args) args)
  (+ 1 2 3))
\end{lstlisting}
\end{figure}

\subsubsection{if}

Для вычисления значения условной формы \texttt{if} происходит вычисление значения условия, на основе которого дальше вычисляется одна из веток -- ветка "<истина"> и ветка "<ложь">. Значением формы является значение вычисленной ветки. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defmacro m (x) (if x 1 2))
  (m t)
  (m nil))
->
(progn
  1
  2)
\end{lstlisting}
\end{figure}

\subsubsection{setq}

Форма \texttt{setq} позволяет создавать и модифицировать глобальные переменные, существующие на уровне макросов. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defmacro init () (setq count 0))
  (init)

  (defmacro m () (setq count (+ count 1)))
  (m)
  (m)
  (m))
->
(progn
  0

  1
  2
  3)
\end{lstlisting}
\end{figure}

\subsubsection{progn}

Форма \texttt{progn} поочерёдно вычисляет данную ей последовательность выражений и возвращает в качестве результата значение последнего выражения. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defmacro m ()
    (progn
      (setq x 1)
      (setq x (+ x 1))))
  (m))
->
(progn
  2)
\end{lstlisting}
\end{figure}

\subsubsection{let}

Форма \texttt{let} принимает на вход список пар символов и значений, вычисляет значения символов, расширяет окружение этими парами и поочерёдно выполняет внутренние выражения с расширенным окружением. По выходу из формы окружение восстанавливается. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defmacro m (x)
    (let ((y (* x 2)))
      (+ y 1)))
  (m 5))
->
(progn
  11)
\end{lstlisting}
\end{figure}

\subsubsection{cond}

Форма \texttt{cond} принимает на вход список пар, левый элемент которых -- условие, а правый -- следствие. Форма последовательно вычисляет значение условия каждой пары по очереди, пока одно из этих условий не вернёт истинное значение, после чего вычисляется и возвращается значение следствия соответствующей пары. Если ни одно из условий не вернуло истину, результатом формы будет \texttt{nil}. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defmacro sign (x)
    (cond
      ((< x 0) -1)
      ((> x 0) 1)
      (t 0)))
  (sign 5))
->
(progn
  1)
\end{lstlisting}
\end{figure}

\subsubsection{quote}

Форма \texttt{quote} возвращает свой аргумент, не вычисляя его. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defmacro hello () '(print "hello"))
  (hello))
->
(progn
  (print "hello"))
\end{lstlisting}
\end{figure}

\subsubsection{backquote}

Форма \texttt{backquote} возвращает свой аргумент, если он атом или пустой список. Если аргумент содержит форму \texttt{comma}, то вычисляется выражение внутри \texttt{comma}, после чего его значение подставляется в результирующее выражение. Если аргумент -- список, первый элемент которого -- форма \texttt{comma-at}, то результат вычисления выражения внутри \texttt{comma-at} объединяется с оставшейся вычисленной частью списка. Если аргумент -- произвольный список, то форма по очереди вычисляет каждый элемент списка. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defun null (x) (eq x ()))
  (defun list-length (list)
    (if (null list)
      0
      (+ (list-length (cdr list)) 1)))
  (defmacro mean (&rest xs)
    `(/ (+ ,@xs) ,(list-length xs)))
  (mean 1 2 3))
->
(progn
  (/ (+ 1 2 3) 3))
\end{lstlisting}
\end{figure}

\subsubsection{function}

Форма \texttt{function} вычисляет и возвращает функциональный объект по символу или лямбда-выражению. Для символа производится поиск функции в списке функций. Для лямбда-выражения производится проверка на правильную структуру лямбда-выражения. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defmacro add (&rest args) (apply #'+ args))
  (add 1 2 3))
->
(progn
  6)
\end{lstlisting}
\end{figure}

\subsubsection{funcall}

Форма \texttt{funcall} выполняет данный ей объект замыкания лямбда-выражения, расширяя при этом окружение данными параметрами. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defmacro m () (funcall #'(lambda () (+ 1 2 3))))
  (m))
->
(progn
  6)
\end{lstlisting}
\end{figure}


\subsection{Фаза 2. Статический анализ}

На данном этапе компиляции входное S-выражение преобразуется в другое абстрактное синтаксическое дерево, состоящее из простых операций и сохраняющее смысл исходного выражения.

\subsubsection{Макросы}

Форма \texttt{defmacro} обрабатывается модулем \texttt{macro}, а в искомое дерево вместо определения макроса попадает форма \texttt{NOP} -- пустой оператор, который означает отсутствие действия и впоследствии будет удалён на следующих этапах компиляции.

При обработке вызова функции в случае, если функция -- макрос, происходит раскрытие макроса в модуле macro и затем происходит анализ полученного выражения.

\subsubsection{Лексическое окружение}

Для статического разрешения переменных используется лексическое окружение. Оно представляет собой набор кадров, каждый из которых содержит список связанных переменных. В начале компиляции окружение пустое. При анализе определения функции окружение расширяется новым кадром, состоящим из аргументов данной функции. По окончании обработки определения функции кадр этой функции удаляется из окружения. Таким образом, происходит симуляция поведения окружения при выполнении программы на этапе компиляции, чтобы заранее определить положения переменных в окружении и избавиться от поиска переменных по их имени во время выполнения программы.

\subsubsection{Константы и переменные}

Константы переносятся в искомое дерево без изменений в форму \texttt{CONST}. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(CONST 86)
\end{lstlisting}
\end{figure}

При анализе переменной определяется её тип. Должно быть определено 3 типа переменных:

\begin{enumerate}
\item Глобальные переменные. Каждая глобальная переменная заносится в отдельный список \texttt{*global-vars*}. Обращение к таким переменным происходит по индексу переменной в этом списке с помощью формы \texttt{GLOBAL-REF}:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(GLOBAL-REF 0) ; получить значение глобальной переменной со смещением 0
\end{lstlisting}
\end{figure}

В начале компиляции в список глобальных переменных добавляются символы \texttt{T} и \texttt{NIL}. С учётом этого, память для глобальных переменных виртуальный машины должна иметь размер как минимум в 2 элемента, первый из которых должен по умолчанию иметь значение символа \texttt{T}, второй -- значение пустого списка. Таким образом, обращение к символу \texttt{T} принимает вид \texttt{(GLOBAL-REF 0)}, а к символу \texttt{NIL} или пустому списку -- \texttt{(GLOBAL-REF 1)}.

\item Локальные переменные -- переменные, определённые в текущем кадре активации. Эти переменные заносятся в текущий кадр лексического окружения, и обращение к ним происходит по смещению в этом кадре:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(LOCAL-REF 0) ; получить значение локальной переменной со смещением в кадре 0
\end{lstlisting}
\end{figure}

\item Свободные переменные -- переменные, определённые в одном из более верхних кадров активации по отношению к текущему. Эти переменные также заносятся в лексическое окружение, и обращение к ним происходит с помощью указания смещения относительно текущего кадра и смещения внутри кадра:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(DEEP-REF 1 0) ; получить значение свободной переменной со смещением кадра 1 и со смещением в кадре 0
\end{lstlisting}
\end{figure}

\end{enumerate}

Таким образом, в результате статического анализа все обращения к переменным будут заменены на смещения, благодаря чему не придётся проводить их поиск во время выполнения программы.

\subsubsection{progn}

Последовательность выражений \texttt{progn} преобразуется в список, в котором первый элемент -- символ \texttt{SEQ}, а остальные -- операции промежуточной формы, соответствующие подвыражениям исходного \texttt{progn} в том же порядке появления. Для \texttt{progn} с единственным подвыражением вместо формы \texttt{SEQ} генерируется только операция для вычисления подвыражения. Пустой \texttt{progn} преобразуется в вычисление пустого списка. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn 1 2 3) -> (SEQ (CONST 1) (CONST 2) (CONST 3))
(progn 1) -> (CONST 1)
(progn) -> (CONST ())
\end{lstlisting}
\end{figure}

\subsubsection{if}

Условная форма \texttt{if} трансформируется в список, первый элемент которого -- символ \texttt{ALTER}, второй -- операция для вычисления условия, третий -- операция для вычисления по первой ветке, четвёртый -- по второй ветке. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(if t 1 2) -> (ALTER (GLOBAL-REF 0) (CONST 1) (CONST 2))
\end{lstlisting}
\end{figure}

\subsubsection{setq}

При анализе присваивания \texttt{setq} определяется тип переменной, затем генерируется соответствующая операция присваивания (\texttt{GLOBAL-SET} для глобальных переменных, \texttt{LOCAL-SET} -- для локальных и \texttt{DEEP-SET} -- для свободных) вместе с рекурсивной генерацией подвыражения. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(setq x 5) -> (GLOBAL-SET 2 (CONST 5))
\end{lstlisting}
\end{figure}

\subsubsection{Функции и замыкания}

Пользовательские функции с фиксированным количеством аргументов обрабатываются аналогично лямбда-выражениям. Имя функции, смещение кадра окружения, количество аргументов сохраняется в таблицу \texttt{*fix-functions*} для последующей проверки при вызове. Для лямбда-функций имена генерируются автоматически.

Функции с переменным числом аргументов сохраняются в таблицу \texttt{*nary-functions*}, в которой хранится имя функции, смещение кадра окружения и число фиксированных аргументов (остальные аргументы добавляются в виде списка).

При обработке формы \texttt{labels} текущее лексическое окружение расширяется определениями локальных функций. Сохраняется имя самой локальной функции и автоматически сгенерированное имя функции. Дальше генерируется код для этих функций, после чего окружение восстанавливается к предыдущему состоянию.

Примитивы с фиксированным числом аргументов хранятся в таблице \texttt{*fix-primitives*}. Хранится имя примитива и число аргументов.

Примитивы с переменным числом аргументов хранятся в таблице \texttt{*nary-primitives*}. Хранится имя примитива и число фиксированных аргументов.

При компиляции применения функции, определяется тип функции: глобальная, лямбда или примитив. Затем происходит проверка числа аргументов. Для лямбда-функций перед вызовом вставляется скомпилированное тело функции. Перед вызовом функции активируется замкнутое окружение функции, создаётся кадр активации с заранее вычисленным размером, вычисляются слева направо аргументы и заносятся в текущий кадр. При вызове примитивов кадр активации не создается.

Для компиляции тела функций используется форма FUNC, которая представляет собой именованную метку, на которую можно совершить переход, что можно использовать для выполнения функций в байт-коде. Для генерации определения функций можно использовать имя функции в качестве названия метки.

Для перехода на метку функции с фиксированным числом аргументов используется форма \texttt{FIX-CALL}, для функции с переменным числом аргументов -- \texttt{NARY-CALL}. Форма \texttt{FIX-CALL} содержит название метки, номер кадра окружения, в котором была определена функция и относительно которой нужно создавать новый кадр, и список операций, каждая из которых вычисляет аргумент функции, в порядке передачи аргументов функции. В форме \texttt{NARY-CALL} дополнительно содержится количество фиксированных аргументов функции.

Для возврата из функции обратно в место после вызова функции используется форма \texttt{RETURN}.

Пример промежуточной формы определения функции через \texttt{defun} и её вызова:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(defun f (x)
  x)
->
(SEQ
  (LABEL F
    (SEQ
      (LOCAL-REF 0)
      (RETURN)))
  (FIX-CALL F 0 ((CONST 5)))) ; переход на метку функции с фиксированным числом аргументов, с переходом на кадр под номером 0 и со списоком аргументов, состоящих из константы 5
\end{lstlisting}
\end{figure}

\paragraph{tagbody}

Форма \texttt{tagbody} преобразуется в форму \texttt{SEQ} с заменой символов на формы \texttt{LABEL} и формы \texttt{go} на \texttt{GOTO} -- безусловный переход на метку. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(tagbody
  (go skip)
  1
  2
  skip
  3)
->
(SEQ
  (GOTO G1)
  (CONST 1)
  (CONST 2)
  (LABEL G1
    (CONST 3)))
\end{lstlisting}
\end{figure}

\subsubsection{catch/throw}

Форма \texttt{catch} преобразуется в аналогичную форму \texttt{CATCH}, и таким же образом форма \texttt{throw} превращается в форму \texttt{THROW}. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(catch 'test
  (throw 'test 5))
->
(CATCH (CONST TEST) (THROW (CONST TEST) (CONST 5)))
\end{lstlisting}
\end{figure}

\subsection{Фаза 3. Оптимизация}

На данном этапе компиляции промежуточное дерево, получившееся в результате статического анализа, преобразуется посредством применения указанных оптимизаций. Оптимизация выражения состоит из последовательного вызова функций, соответствующих указанным флагам оптимизации.


\subsubsection{Оптимизация тривиальных условий (trivial-condition)}

Форму \texttt{ALTER}, условие которой -- константа, можно упростить, вычислив условие на этапе компиляции и заменив всю форму одной из её веток. Примеры:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(ALTER (CONST T)
  (CONST 1)
  (CONST 2))
->
(CONST 1)

(ALTER (CONST NIL)
  (CONST 1)
  (CONST 2))
->
(CONST 2)
\end{lstlisting}
\end{figure}


\subsubsection{Оптимизация арифметических выражений (simplify-arithmetic)}

Заменяет примитивы с произвольным числом аргументов на примитивы с двумя аргументами. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(+ 1 2 3 4)
->
(+ (+ (+ 1 2) 3) 4)
\end{lstlisting}
\end{figure}


\subsubsection{Удаление неиспользуемого кода (dead-code-elimination)}

Включает в себя несколько оптимизаций.


\paragraph{Оптимизация загрузки регистра аккумулятора}

Последовательности идущих подряд обращений к константам или переменным, встречающиеся в формах \texttt{SEQ}, преобразуются таким образом, что все, кроме последнего обращения в последовательности, а также обращения перед командой \texttt{RETURN}, удаляются. К таким обращениям относятся \texttt{CONST}, \texttt{GLOBAL-REF}, \texttt{LOCAL-REF} и \texttt{DEEP-REF}. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(SEQ
  (GLOBAL-SET 2 (CONST 5))
  (CONST 10)
  (GLOBAL-REF 2)
  (GLOBAL-REF 1))
->
(SEQ
  (GLOBAL-SET 2 (CONST 5))
  (GLOBAL-REF 1))
\end{lstlisting}
\end{figure}

Данная оптимизация также удаляет операции обращения к переменным, если непосредственно перед такой операцией находится присваивание этой же переменной, так как результат вычисления операции присваивания равен значению, которое было присвоено переменной в ходе этой операции, поэтому последующее обращение к этой переменной избыточно. К операциям присваивания относятся \texttt{GLOBAL-SET}, \texttt{LOCAL-SET} и \texttt{DEEP-SET}. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(SEQ
  (GLOBAL-SET 2 (CONST 5))
  (GLOBAL-REF 2))
->
(SEQ
  (GLOBAL-SET 2 (CONST 5)))
\end{lstlisting}
\end{figure}


\paragraph{Удаление функций после подстановки}

В первой фазе собирается информация о возможности подстановки функций. Во второй фазе для функций, которые вызываются один раз и могут быть подставлены, удаляется узел \texttt{LABEL}. Но удаление невозможно, если:

\begin{itemize}
\item \texttt{LABEL} расположен внутри \texttt{FIX-CLOSURE} (замыкание lambda-функции);
\item замыкание пользовательской функции - \texttt{FIX-CLOSURE} без \texttt{LABEL} (для этого необходимо собрать информацию о замыкании функции в первом проходе).
\end{itemize}


\subsubsection{Оптимизация хвостовой рекурсии (tail-call)}

Хвостовые вызовы \texttt{TAIL-CALL} и \texttt{TAIL-NCALL} не должны увеличивать число кадров активации, то есть они используют текущий кадр, изменяя параметры вызова в нем. При хвостовом вызове не нужно также сохранять и восстанавливать текущий кадр активации, поэтому хвостовой вызов транслируется в \texttt{JMP}.

Чтобы найти вызовы из хвостовых позиций в функцию \texttt{inner-compile} добавляется еще один логический аргумент \texttt{tail}, который означает, расположено ли текущее выражение в хвостовой позиции.

Основные правила определения хвостовой позиции:

\begin{enumerate}
\item Форма \texttt{(setq x e)}. Выражение \texttt{e} -- не хвостовой вызов, потому что оно записывается в переменную, то есть не является последним действием текущего блока.
\item Форма \texttt{(if e t f)}. Выражение \texttt{e} -- не хвостовой вызов, т.к. после его вычисления следует вычисление одной из веток условия.
\item Форма \texttt{(progn $x_1$ ... $x_n$)}. Все выражения, кроме последнего ($x_n$) -- не хвостовые вызовы.
\item Применение функции -- всегда хвостовой вызов для тела функции.
\end{enumerate}


\subsubsection{Свёртка констант (constant-folding)}

Константные выражения считаются на этапе компиляции. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(setq x (+ 1 2 3)) -> (setq x 6)
\end{lstlisting}
\end{figure}

Сворачиваются только примитивы, не имеющие побочных эффектов, кроме \texttt{make-array} и \texttt{make-string}.


\subsubsection{Удаление неиспользуемых функций (unused-functions)}

В первой фазе компиляции для всех функций (глобальных и локальных) подсчитывается число ссылок на них. Ссылка может быть как вызов функции (применение функции) или как создание замыкания (объекта-функции). Если у функции 0 ссылок, значит она не используется в коде, и ее можно удалить.


\subsubsection{Встраивание функций, которые вызываются один раз (beta-expansion)}

Происходит во второй фазе оптимизации, потому что тела функций могут быть переопределены.

Встраивание возможно, когда:

\begin{itemize}
\item функция не рекурсивная;
\item нет установки локальных переменных (\texttt{LOCAL-SET});
\item к каждому аргументу идет обращение только один раз;
\item в выражении аргументов нет побочных эффектов;
\item в функции есть замыкание, в теле которого нет \texttt{DEEP-REF};
\item нет повторного определения функции.
\end{itemize}

Вместо вызова функции можно подставить скомпилированное тело функции. В теле функции необходимо заменить обращения к переменным:

\begin{itemize}
\item локальные переменные (\texttt{LOCAL-REF}) заменяются на скомпилированные выражения аргументов;
\item свободные переменные понижаются в уровне обращения (\texttt{DEEP-REF/SET 2} заменяется на \texttt{DEEP-REF/SET 1}, \texttt{DEEP-REF/SET 1} заменяется на \texttt{LOCAL-REF/SET}) (не внутри \texttt{FIX-LET});
\item \texttt{RETURN} заменяется на \texttt{NOP}.
\end{itemize}

При подстановке внутри замыканий (\texttt{FIX-CLOSURE}) необходимо учитывать относительный уровень глубины вызова.

Для того, чтобы была возможна каскадная подстановка, новое оптимизированное тело сохраняется.

Лямбда функции \texttt{FIX-LET} всегда вызываются один раз, и их тоже можно встроить по тем же условиям.

Встраивание функций с переменным числом аргументов аналогично встраиванию обычных функций за исключением того, что необходимо собрать необязательные аргументы в список в месте подстановки.


\subsubsection{Полное встраивание функций (all-inline)}

Аналогично оптимизации \texttt{beta-expansion} за исключением того, что разрешается также встраивать те функции, которые вызываются более одного раза.

Для элементов \texttt{CONST}, \texttt{GLOBAL-REF}, \texttt{LOCAL-REF}, \texttt{DEEP-REF} и \texttt{RETURN} существуют соответствующие инструкции, поэтому они генерируются как инструкции с мнемониками и операндами без изменений.

Для элементов \texttt{GLOBAL-SET}, \texttt{LOCAL-SET} и \texttt{DEEP-SET} сначала генерируется вычисление их аргументов, при этом идёт симуляция глобальных переменных и окружения, чтобы затем эти инструкции можно было сгенерировать с правильными аргументами.

Элемент \texttt{FUNC} используется для генерации тела функции, поэтому сначала рассчитывается метка после тела функции, генерируется переход на эту метку, после этого генерируется тело самой функции, и в конце добавляется эта метка.

В элементе \texttt{SEQ} для каждого дочернего элемента рекурсивно по очереди происходит дальнейшая генерация.

Для элемента \texttt{ALTER} рассчитываются метки для ветки по лжи и для конца if-блока, затем генерируется условие, условный переход на ветку по лжи, тело по истине и безусловный переход на конец блока, метка и тело по лжи, и метка конца блока.

В элементе \texttt{FIX-PRIM} для каждого аргумента генерируется вычисление этого аргумента и инструкция \texttt{PUSH}, затем генерируется вызов соответствующего примитива \texttt{PRIM}.

В элементе \texttt{NARY-PRIM} для каждого аргумента генерируется вычисление этого аргумента и инструкция \texttt{PUSH}, необязательные аргументы собираются в список с помощью команды \texttt{PACK} (удаляются из стека и добавляются как список), затем генерируется вызов соответствующего примитива \texttt{NPRIM}.

Для элемента \texttt{REG-CALL} генерируется вычисление каждого аргумента и добавление в стек, установление соответствующего окружения для текущей функции (\texttt{SET-ENV}), создание кадра активации и добавления в окружение аргументов функции из стека (\texttt{ALLOC}), вызов самой функции (\texttt{REG-CALL}) и в конце восстановление окружения (\texttt{RESTORE-ENV}).

Для элемента \texttt{APPLY} генерируется вычисление каждого аргумента и добавление в стек через \texttt{PUSH}, опциональное объединение этих аргументов в список с помощью \texttt{PACK}, проверка функционального объекта в аккумуляторе на примитив, условный переход на вызов функции, вызов примитива, безусловный переход на конец блока, метка для вызова функции, сохранение текущего кадра окружения в стек, вызов функции, восстановление окружения и метка конца блока.


\subsection{Фаза 5. Ассемблер}

Последним шагом компиляции является генерация байт-кода из ассемблер-кода. Ассемблирование происходит в 2 прохода.

При первом проходе мнемоники опкодов заменяются соответствующими байтами опкодов, при этом если встречается метка, то она не добавляется в результирующий байт-код, но запоминается в хеш-таблицу c адресом следующей инструкции, и если встречается инструкция перехода, то в отдельный список добавляется текущий адрес с меткой, которую использует данная инструкция.

Второй проход идёт по списку адресов с метками и в байт-коде заменяется соответствующие метки на адреса меток из хеш-таблицы.

На выходе компилятора:

\begin{itemize}
\item список констант;
\item число глобальных переменных;
\item массив с байт-кодом.
\end{itemize}

Точку входа в программу можно посчитать (перейти по всем \texttt{JMP}, до первого оператора не \texttt{JMP}).
