\section{Техническое задание}
\subsection{Основание для разработки}

Основанием для разработки является задание на выпускную квалификационную работу бакалавра "<Разработка программной системы компилятора функционального языка программирования Common Lisp с оптимизациями">.

\subsection{Цель и назначение разработки}

Основной задачей выпускной квалификационной работы является разработка компилятора Common Lisp с оптимизациями и его тестирование.

Целью разработки является создание оптимизирующего компилятора Common Lisp.

Задачами данной разработки являются:
\begin{itemize}
\item изучение процесса компиляции языков программирования и функциональных языков в частности;
\item изучение методов оптимизации компиляторов;
\item проектирование компилятора с оптимизациями;
\item реализация компиляции S-выражения в промежуточную форму;
\item реализация генерации кода ассемблера;
\item реализация генерации байт-кода;
\item реализация оптимизации промежуточной формы;
\item реализация генерации машинного кода.
\end{itemize}

\subsection{Реализуемое подмножество Common Lisp}

\subsubsection{S-выражения}

Компилируемое выражение представляет собой S-выражение -- атом (символ, число, строка и др.) или список из нуля или более S-выражений.

\subsubsection{Типы объектов}

К объектам относятся числа: \texttt{5}, \texttt{0xf}, \texttt{3.14}, символы: \texttt{AB}, списки: \texttt{(1 2 3)}, пары: \texttt{(1 . 2)}, строки: \texttt{"123"}, массивы: \texttt{\#(1 2 3)}, одиночные символы: \texttt{\#A}, функции: \texttt{\#'(lambda (x) x)}. При переводе в логический тип \texttt{NIL} считается как ложь, всё остальное -- истина (при выводе истины используется символ \texttt{T}).

\subsubsection{Специальные формы}

\begin{enumerate}
\item \texttt{(quote x)} -- особый оператор, который возвращает свой аргумент, не вычисляя его. Сокращенная запись -- \texttt{'x} (кавычка \texttt{x}).
\item \texttt{(if <условие> <ветка true> <ветка false>)}. Условный оператор, считающий в своём условии истиной всё, кроме \texttt{nil}.
\item \texttt{(setq <переменная$_1$> <выражение$_1$> ... <переменная$_n$> <выражение$_n$>)}. Создает глобальные переменные и присваивает им значения выражений. Возвращает значение последнего выражения.
\item \texttt{(progn $e_1$ $e_2$ ... $e_n$)}. Последовательно вычисляет выражения $e_1$ ... $e_n$. Возвращает значение последнего выражения $e_n$.
\item \texttt{(tagbody <список символов или выражений>)}. Последовательно вычисляет вложенные выражения. Символы выступают в роли меток, на которые можно перейти с помощью оператора \texttt{go}. Возвращает \texttt{nil}.
\item \texttt{(catch <имя> <список выражений>)}. Блок с возможностью нелокального выхода. Сначала вычисляется имя блока, затем последовательно вычисляются выражения. Если встречается \texttt{(throw <имя блока> <результат вычисления блока>)}, то вычисляется имя блока и происходит выход из ближайшего блока с этим именем. Если выход не происходит, то возвращается результат последнего выражения. Все имена вычисляются динамически.
\item \texttt{(labels (<список функций>) <список выражений>)}. Объявляет локальные функции (в том числе и рекурсивные), после чего последовательно вычисляет выражения.
\end{enumerate}

\subsubsection{Примитивы}

\begin{itemize}

\paragraph{Списки}

\item \texttt{(car x)} -- возвращает первый элемент списка (левый объект пары);
\item \texttt{(cdr x)} -- возвращает все кроме первого элемента списка (правый объект пары);
\item \texttt{(cons x y)} -- создает точечную пару с объектами \texttt{x} и \texttt{y};
\item \texttt{(rplaca x y)} -- заменяет CAR списка \texttt{x} на значение \texttt{y};
\item \texttt{(rplacd x y)} -- заменяет CDR списка \texttt{x} на значение \texttt{y};
\item \texttt{(list $e_1$ $e_2$ ... $e_n$)} -- создаёт список со значениями выражений $e_1$ $e_2$ ... $e_n$.

\paragraph{Арифметические операции}

\item \texttt{(+ x y)} -- сложение \texttt{x} и {y};
\item \texttt{(- x y)} -- вычитание \texttt{x} и {y};
\item \texttt{(* x y)} -- умножение \texttt{x} и {y};
\item \texttt{(/ x y)} -- деление \texttt{x} и {y};
\item \texttt{(sin num)} -- синус вещественного числа \texttt{num};
\item \texttt{(cos num)} -- косинус вещественного числа \texttt{num}.

\paragraph{Предикаты}

\item \texttt{(atom x)} -- возвращает \texttt{T} (истина), если аргумент является атомом, иначе возвращает \texttt{NIL}, эквивалентный пустому списку \texttt{()}, что означает ложь;
\item \texttt{(eq x y)} -- возвращает \texttt{T}, если \texttt{x} и \texttt{y} -- один и тот же объект, иначе возвращает \texttt{NIL};
\item \texttt{(and <список условий>)} -- логическая операция И;
\item \texttt{(or <список условий>)} -- логическая операция ИЛИ;
\item \texttt{(equal x y)} -- сравнение \texttt{x} и \texttt{y};
\item \texttt{(> x y)} -- \texttt{x} больше чем \texttt{y};
\item \texttt{(< x y)} -- \texttt{x} меньше чем \texttt{y};
\item \texttt{(\% x y)} -- остаток деления \texttt{x} и \texttt{y};
\item \texttt{(pairp x)} -- предикат, является ли \texttt{x} парой;
\item \texttt{(symbolp x)} -- предикат, является ли \texttt{x} символом;
\item \texttt{(integerp x)} -- предикат, является ли \texttt{x} числом;
\item \texttt{(stringp x)} -- предикат, является ли \texttt{x} строкой;
\item \texttt{(arrayp x)} -- предикат, является ли \texttt{x} массивом;
\item \texttt{(floatp x)} -- предикат, является ли \texttt{x} числом с плавающей запятой.

\paragraph{Побитовые операции}

\item \texttt{(\& x y)} -- побитовое И от \texttt{x} и \texttt{y};
\item \texttt{(bitor x y)} -- побитовое ИЛИ от \texttt{x} и \texttt{y};
\item \texttt{(<{<} x y)} -- сдвиг влево \texttt{x} на \texttt{y};
\item \texttt{(>{>} x y)} -- сдвиг вправо \texttt{x} на \texttt{y}.

\paragraph{Строки и символы}

\item \texttt{(intern str)} -- создаёт символ на основе строки \texttt{str};
\item \texttt{(symbol-name sym)} -- создаёт строку на основе символа \texttt{sym};
\item \texttt{(string-size str)} -- возвращает длину строки \texttt{str};
\item \texttt{(concat $s_1$ ... $s_n$)} -- объединяет строки в одну общую строку (возвращает пустую строку при отсутствии аргументов);
\item \texttt{(inttostr x)} -- перевод целочисленного числа \texttt{x} в строку;
\item \texttt{(code-char x)} -- создает символ по коду \texttt{x};
\item \texttt{(char-code c)} -- возвращает код символа \texttt{c};
\item \texttt{(make-string len c)} -- создаёт строку длиной \texttt{len}, заполненную символом \texttt{c};
\item \texttt{(char str i)} -- получает символ строки \texttt{str} по индексу \texttt{i};
\item \texttt{(sets str i c)} -- заменяет символ в строке \texttt{str} с индексом \texttt{i} символом \texttt{c};
\item \texttt{(subseq str a b)} -- получение подстроки из строки \texttt{str} (от индекса \texttt{a} до \texttt{b}).

\paragraph{Массивы}

\item \texttt{(make-array x)} -- создание пустого массива длины \texttt{x};
\item \texttt{(array-size x)} -- возвращает размер массива \texttt{x};
\item \texttt{(aref x i)} -- чтение элемента массива \texttt{x} по индексу \texttt{i};
\item \texttt{(seta x i y)} -- присвоение значения \texttt{y} элементу \texttt{i} массива \texttt{x}.

\paragraph{Печать объектов}

\item \texttt{(print x)} -- печать объекта \texttt{x};
\item \texttt{(putchar c)} -- печать символа \texttt{c}.

\paragraph{Другие примитивы}

\item \texttt{(funcall <функция> <аргументы>)} -- вызывает функцию с аргументами;
\item \texttt{(apply <функция> <список аргументов>)} -- вызывает функцию с аргументами, переданными через список;
\item \texttt{(eval <выражение>)} -- вычисляет значение выражения как ещё одно выражение;
\item \texttt{(error msg)} -- останавливает вычисление, выводит сообщение об ошибке и возвращается в REPL цикл.

\end{itemize}

\subsubsection{Лямбда выражения}

Лямбда выражение -- это анонимная функция \texttt{\#'(lambda ($p_1$ ... $p_n$) $e_1$ ... $e_n$)}, где $p_1$ ... $p_n$ -- это параметры функции, а $e_1$ ... $e_n$ -- выражения тела функции.

Вызов функции -- это следующее выражение:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(funcall #'(lambda (p1 ... pn) e1 ... en) a1 ... an)
\end{lstlisting}
\end{figure}

Сначала вычисляются все аргументы $a_1$ ... $a_n$. Затем каждому параметру $p_1$ ... $p_n$ ставится в соответствие вычисленное значение аргументов $a_1$ ... $a_n$. После этого последовательно вычисляются выражения $e_1$ ... $e_n$, содержащие параметры, вместо которых будут подставлены их значения.

\subsubsection{Определение функций}

Новую функцию можно создать с помощью оператора \texttt{defun}:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(defun null (x)
  (eq x ()))
\end{lstlisting}
\end{figure}

\subsubsection{Глобальные переменные}

Глобальные переменные существуют все время работы. Они создаются с помощью функции \texttt{(defvar имя\_переменной [значение])}. Значение может быть выражением:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(defvar a 10) -> A
A -> 10
\end{lstlisting}
\end{figure}

При отсутствии значения в переменную записывается значение \texttt{NIL}.

Установить значение переменной можно с помощью функции \texttt{setq} (особая форма). Если такой переменной не было то она создается:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(setq x 1) -> 1
x -> 1
\end{lstlisting}
\end{figure}

Можно одной функцией установить значения нескольких переменных:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
  (setq a 1 b 2 c 3)
\end{lstlisting}
\end{figure}

Если переменная локальная (параметр функции), то \texttt{setq} ее модифицирует:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(defun test(x)
  (setq x 10)) ; модификация параметра
\end{lstlisting}
\end{figure}

Глобальные переменные являются динамическими, то есть их значение используется всегда последнее, которое было связано.

\subsubsection{Макросы}

Макрос задает шаблон для генерации выражения.

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(defmacro test (var val)
  (list 'setq var val))
\end{lstlisting}
\end{figure}

При вызове макроса сначала происходит вычисление тела макроса (развертывание макроса):

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(test abc 100) -> (setq abc 100)
\end{lstlisting}
\end{figure}

Затем получившееся выражение вычисляется:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(setq abc 100) ; abc = 100
\end{lstlisting}
\end{figure}

Обратная кавычка вычисляется как обычное цитирование, но также позволяет указывать, какие части цитирования должны быть вычислены. Эти части указываются с помощью запятой:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(setq a 10)
`(a b c ,a) -> (A B C 10)
\end{lstlisting}
\end{figure}

Запятая-at (\texttt{,@}) служит для того, чтобы подставить список (результат вычисления выражения внутри запятой-at должен быть списком):

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(defvar a '(1 2 3)) -> (1 2 3)
`(,a ,@a) -> ((1 2 3) 1 2 3)
\end{lstlisting}
\end{figure}

Можно посмотреть результат макроподстановки с помощью функции \texttt{macroexpand}:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(macroexpand '(if (= 1 1) 2 3)) -> (COND ((= 1 1) 2) (T 3))
\end{lstlisting}
\end{figure}

\subsubsection{Функции как объекты первого класса}

Функции можно передавать в качестве параметров других функций и возвращать функции. Типы функций:

\begin{itemize}
\item локальные;
\item глобальные;
\item встроенные;
\item лямбда.
\end{itemize}

Пространство имен функций не совпадает с пространством имен переменных. Функция передается в качестве параметра как лямбда-выражение или символ. Но при вызове функции из параметра мы должны указать, что имя символа (переменная) относится к пространству имен функций. Для вызова функций по параметру используется примитив \texttt{funcall}. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
; вызов примитива
(funcall #'+ 1 2 3) -> 6

; вызов lambda
(funcall #'(lambda (x y) (+ x y)) 2 3) -> 5

; вызов пользовательской функции по символу
(defun test (x) (+ x 1))
(funcall #'test 4) -> 5
\end{lstlisting}
\end{figure}

Оператор \texttt{\#'} является сокращением специальной формы \texttt{function}, создающей объект-замыкание, в котором сохраняется текущее окружение. При применении функции-замыкания будет использоваться не текущее окружение, а окружение, в котором было создано замыкание. Свободные локальные переменные являются лексическими, то есть сохраняют свое значение, которое было на момент замыкания. Глобальные переменные ведут себя динамическим образом -- берется их значение в момент вызова функции.

\subsubsection{Обработка ошибок}

Обработка исключения для выражения:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(handle <выражение>
  (error (<параметры исключения>)
    <тело обработчика любых исключений в том числе просто error>)
  (division-by-zero (c)
    <тело обработчика деления на ноль>)
  ...
  (<пользовательское исключение> (<параметры>)
    <обработчик пользовательского исключения>))
\end{lstlisting}
\end{figure}

Для \texttt{error} существует обработчик по умолчанию, который выводит сообщение об ошибке. Примеры ошибок: \texttt{division-by-zero} (деление на ноль), \texttt{index-error} (выход за границы массива). Исключение может быть внутреннее или пользовательское:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(handle (if nil (raise 'my-error "My error" 20 30))
  (my-error (params)
  (print params)))
\end{lstlisting}
\end{figure}

\subsection{Программный интерфейс оптимизирующего компилятора}

Программный интерфейс компилятора должен состоять из единственной функции -- \texttt{(compile expr optimization-flags)}, которая принимает на вход исходное S-выражение и флаги оптимизации \texttt{optimization-flags} и преобразует его в эквивалентный байт-код для выполнения на виртуальной машине.

Флаги оптимизации \texttt{compiler-optimizations} должны быть в виде списка символов, каждый из которых включает ту или иную оптимизацию компиляции. Пустой список выключает все оптимизации, специальный символ \texttt{all} -- включает все доступные оптимизации.

Результатом функции является список, первый элемент которого -- массив инструкций для целевой виртуальной машины, второй -- список констант, используемых в программе, и третий -- размер памяти для глобальных переменных. В случае ошибки компиляции вызывается исключение.


\subsubsection{Флаги оптимизации компилятора}

Функция \texttt{compile} должна принимать следующие флаги оптимизации:


\paragraph{trivial-condition}

\texttt{trivial-condition} -- оптимизация тривиальных условий. Пример:
\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(if t 1 2) -> 1
\end{lstlisting}
\end{figure}


\paragraph{simplify-arithmetic}

\texttt{simplify-arithmetic} -- оптимизация арифметических выражений (замена примитивов с произвольным числом аргументов на примитивы с двумя аргументами). Пример:
\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(+ 1 2 3 4) -> (+ (+ (+ 1 2) 3) 4)
\end{lstlisting}
\end{figure}


\paragraph{dead-code-elimination}

\texttt{dead-code-elimination} -- флаг, включающий сразу несколько оптимизаций. К ним относятся:

\begin{enumerate}

\item Оптимизация загрузки регистра аккумулятора

Удаление подряд идущих обращений к константам или переменным, которые не влияют на результат вычисления выражения. Пример:
\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  1
  2
  (setq a 5)
  a
  3
  4)
->
(progn
  (setq a 5)
  4)
\end{lstlisting}
\end{figure}


\item Удаление функций после подстановки 

Удаление объявлений функций, которые тело которых можно подставить во все места их вызова и для которых не создаётся замыкание. Пример:
\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defun f (x) x)
  (print (f 5)))
->
(progn
  (print 5))
\end{lstlisting}
\end{figure}

\end{enumerate}


\paragraph{tail-call}

\texttt{tail-call} -- оптимизация хвостовой рекурсии. Заменяет хвостовой рекурсивный вызов на безусловный переход, благодаря чему функция из рекурсивной превращается в итеративную. Например, следующая рекурсивная функция факториала:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(defun fact (n acc)
  (if (= n 1)
      acc
      (fact (-- n) (* n acc))))
\end{lstlisting}
\end{figure}

использует хвостовую рекурсию, так что при включенной оптимизации данная функция не будет заполнять стек при каждом рекурсивном вызове.


\paragraph{constant-folding}

\texttt{constant-folding} -- свёртка констант. Рекурсивно вычисляет значения примитивов, не имеющих побочных эффектов, с константными аргументами. Например:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(print (+ (5 * 2) 1)) -> (print 11)
\end{lstlisting}
\end{figure}


\paragraph{unused-functions}

\texttt{unused-functions} -- удаление неиспользуемых функций. Удаляет объявления функций, которые ни разу не используются. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defun f1 () 1)
  (defun f2 () 2)
  (defun f3 () 3)
  (print (f3)))
->
(progn
  (defun f3 () 3)
  (print (f3)))
\end{lstlisting}
\end{figure}


\paragraph{beta-expansion}

\texttt{beta-expansion} -- встраивание функций. Подставляет тело функции вместо её вызова в случае, если эта функция вызывается единожды и удовлетворяет ряду требований, гарантирующих, что подстановка не изменит логику программы. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defun f (x) x)
  (print (f 5)))
->
(progn
  (defun f (x) x)
  (print 5))
\end{lstlisting}
\end{figure}


\paragraph{all-inline}

\texttt{all-inline} -- полное встраивание функций. Более агрессивная версия встраивания функций. Подставляет тело функции вместо её вызова независимо от числа вызовов данной функции в случае, если функция удовлетворяет выше упомянутому ряду требований возможности подстановки. Пример:

\begin{figure}[H]
\begin{lstlisting}[language=Lisp]
(progn
  (defun f (x) x)
  (print (f 5))
  (print (f 10))
  (print (f 15)))
->
(progn
  (defun f (x) x)
  (print 5)
  (print 10)
  (print 15))
\end{lstlisting}
\end{figure}



\subsection{Требования к оформлению документации}

Разработка программной документации и программного изделия должна производиться согласно ГОСТ 19.102-77 и ГОСТ 34.601-90. Единая система программной документации.
